# 接口约定

## 当前接口状态

当前阶段只保留初始化框架接口，不实现正式电力巡检业务协议。现有接口主要用于文本指令、任务事件、语音播报、系统状态和移动端调试入口。

正式 `/inspection/*` 任务协议、告警协议和巡检记录协议仍在规划中，当前未实现。

## /inspection_ai/* 话题说明

| 话题 | 类型 | 当前含义 |
|---|---|---|
| `/inspection_ai/text_command` | `std_msgs/String` | 文本任务入口，可由 UI、CLI、语音路由或 mobile bridge 发布 |
| `/inspection_ai/task_event` | `ylhb_interfaces/TaskEvent` | 初始化阶段的通用巡检任务事件 |
| `/inspection_ai/task_status` | `ylhb_interfaces/TaskStatus` | 任务执行状态占位，供后续执行层回传 |
| `/inspection_ai/say_text` | `ylhb_interfaces/SayText` | 语音播报请求 |
| `/inspection_ai/voice_status` | `ylhb_interfaces/VoiceStatus` | TTS 或语音链路状态 |
| `/inspection_ai/task_context_status` | `std_msgs/String` | LLM 任务层上下文 JSON |
| `/inspection_ai/system_command` | `std_msgs/String` | UI 发出的系统控制命令 JSON |
| `/inspection_ai/system_status` | `std_msgs/String` | system supervisor 状态 JSON |
| `/inspection_ai/system_mode` | `std_msgs/String` | `ready`、`mapping`、`fault` 等系统模式占位 |
| `/inspection_ai/voice_command_event` | `std_msgs/String` | 连续语音识别事件 JSON |
| `/inspection_ai/voice_session_status` | `std_msgs/String` | 连续语音会话状态 JSON |

## TaskEvent 当前临时含义

`TaskEvent` 当前复用于初始化阶段任务事件：

| 字段 | 当前解释 |
|---|---|
| `task_id` | 临时任务事件 ID |
| `intent` | `start_inspection`、`pause_inspection`、`resume_inspection`、`cancel_inspection`、`manual_takeover`、`inspect_checkpoint`、`emergency_stop`、`inspection_query` 等 |
| `item_id` | 临时目标 ID，可表示路线、检查点或其他目标 |
| `item_name` | 临时目标名称 |
| `destination` | 临时目的地描述 |
| `confidence` | 规则或 LLM 解析置信度 |
| `source` | `ui`、`voice`、`cli`、`service` 等 |
| `requires_ack` | 是否需要后续执行层确认 |
| `raw_json` | 初始化阶段的扩展 JSON，后续可迁移到正式消息 |

## TaskStatus 当前含义

`TaskStatus` 当前只作为后续执行层回传状态的占位消息。初始化阶段不定义完整状态机，不要求已经覆盖巡检任务生命周期。

## /patrol/* 本地巡逻接口

`ylhb_mobile_bridge` 中的巡逻执行器直接调用 Nav2，路线文件和实机操作流程见
`src/PROJECT_DOC_zh.md`。本节只记录 ROS 话题契约。

| 话题 | 类型 | 当前含义 |
|---|---|---|
| `/patrol/command` | `std_msgs/String` | 接收 `start`、`pause`、`resume`、`cancel`、`reload`、`initialize`、`relocalize`，也支持带 `route_id` 的 JSON |
| `/patrol/status` | `std_msgs/String` | 发布路线、目标点、循环次数、状态和错误信息 JSON |
| `/patrol/event` | `std_msgs/String` | 发布初始位姿、路线开始、到点、任务完成、返航和失败等事件 JSON |

路线到达目标点后，执行器还会向 `/inspection_ai/text_command` 发布到点文本。

## SayText 当前含义

`SayText` 用于向语音输出节点提交播报文本。初始化阶段常用于任务收到、暂停、取消、人工接管等提示，是否真正播报取决于 `enable_tts` 和设备配置。

## raw_json 临时用途

`raw_json` 允许在不新增正式巡检消息的情况下携带扩展字段，便于 PC 初始化阶段调试。后续当任务协议稳定后，应再新增正式消息，而不是长期依赖自由 JSON。

## ylhb_mobile_bridge HTTP API 初始化占位

`ylhb_mobile_bridge` 当前是移动端和 ROS 2 之间的调试桥，不是正式巡检后台。
调试端 APP 仅用于低速底盘控制、建图、导航调试，不承载正式巡检任务流程。

| API | 当前用途 |
|---|---|
| `GET /api/status` | 返回机器人基础状态 |
| `POST /api/cmd_vel` | 发布短时 `/cmd_vel`，用于低速调试 |
| `POST /api/text_command` | 发布文本到 `/inspection_ai/text_command` |
| `POST /api/task` | 将通用任务字段转换为文本后发布到 `/inspection_ai/text_command` |
| `POST /api/stop` | 停止运动并发布停止任务文本 |
| `GET /api/debug/status` | 返回节点和话题调试状态 |
| `POST /api/debug/chassis/test` | 低速底盘测试 |
| `POST /api/debug/chassis/stop` | 停止底盘 |
| `POST /api/debug/mapping/start` | 通过脚本启动建图 |
| `POST /api/debug/mapping/save` | 调用地图保存 |
| `POST /api/debug/mapping/stop` | 停止 bridge 启动的建图进程 |
| `POST /api/debug/navigation/start` | 通过脚本启动导航 |
| `POST /api/debug/navigation/set_initial_pose` | 发布 `/initialpose` |
| `POST /api/debug/navigation/goal` | 发送 Nav2 单点目标 |
| `POST /api/debug/navigation/cancel` | 取消当前 Nav2 目标 |
| `WebSocket /ws/status` | 推送状态 |

### 状态字段说明

`/api/status` 与 `/api/debug/status` 返回的字段对接调试端 APP：

- `system_mode`：来自 `/inspection_ai/system_mode`，取 `ready`/`mapping`/`fault`/`sleep` 等，TRANSIENT_LOCAL QoS。
- `last_map_age_sec`：`/map` 话题最近一次发布距今秒数，建图调试时判断地图是否在持续生成。
- `scan_range_min` / `scan_range_max`：`/scan` 最近一帧的测距范围（米），用于确认雷达数据质量。
- `task_status`：当前回显最后一次通过 bridge 下发的文本命令，不订阅正式任务状态。
- `battery_percent`：电力巡检机器人电量，当前由底盘侧提供；未提供时为空。

`DebugStatus.nodes` 当前检测以下键：`zlac8015d_canopen_controller`、
`slam_toolbox`、`bt_navigator`、`controller_server`、`planner_server`、`amcl`、
`map_server`、`bringup`（检测 `robot_state_publisher`）、`rplidar_node`、
`tf`（检测 `/tf` 话题是否有发布者）。

### 零售遗留字段

调试端 APP 历史版本来自零售场景，以下字段在电力巡检机器人上不实现，
APP 侧对缺失值显示"未知"，属预期行为：

- `salesDialogueStatus`：销售对话状态，电力巡检无此业务。
- `cart`：购物车状态，电力巡检无此业务。

`POST /api/task` 的字段仅为后续扩展占位：

```json
{
  "command": "start",
  "text": "开始巡检任务",
  "task_id": "demo_task",
  "route_id": "route_demo",
  "checkpoint_id": "cp_001",
  "metadata": {}
}
```

## 规划中的 /inspection/* 话题

以下为后续规划，当前未实现：

| 规划话题 | 规划用途 |
|---|---|
| `/inspection/task_command` | 正式任务下发、暂停、恢复、取消、人工接管 |
| `/inspection/task_status` | 正式巡检任务状态 |
| `/inspection/checkpoint_result` | 检查点检测结果 |
| `/inspection/alarm` | 路线级或检查点级告警 |
| `/inspection/record` | 巡检记录摘要 |

当前不要新增 `InspectionTask`、`InspectionAlarm`、`CheckpointResult`、`InspectionRecord` 等正式消息，等业务模型稳定后再设计。
