# 项目概览

## 项目背景

本项目从原智能机器人仓库迁移到电力行业巡检机器人方向，面向电力实训、变电站室外巡检和移动机器人综合开发场景。当前阶段重点不是完成完整业务闭环，而是先把仓库身份、文档、话题命名和 ROS 2 可复用框架整理干净。

## 当前阶段定位

当前交付为 PC 主机端初始化框架：

- 已完成仓库迁移和项目身份切换。
- 已保留 ROS 2 底盘、导航、感知、语音、LLM、UI 和 mobile bridge 框架。
- 已把任务入口切换到 `/inspection_ai/*`。
- 尚未在 Jetson/开发板上实机运行验证。
- 尚未实现正式电力巡检业务闭环。

## 初始化目标

- 清理旧业务文档和命名，建立电力巡检方向的中文文档结构。
- 保留机器人通用能力，便于后续上板调试。
- 保留 LLM 层框架，后续可扩展为巡检任务解析、任务事件和语音提示。
- 保留感知框架，后续接入路线级和检查点级检测。
- 保留 mobile bridge，后续对接手机端或 Web 前端。

## 保留的 ROS 2 通用框架

| 包 | 当前职责 |
|---|---|
| `ylhb_base` | 底盘控制、CAN 配置、IMU、URDF、EKF、SLAM、Nav2 |
| `ylhb_perception` | ZED 输入、YOLO/TensorRT 检测框架、深度定位入口 |
| `ylhb_llm` | 巡检任务事件脚手架、语音输入、语音路由、TTS、UI、system supervisor |
| `ylhb_mobile_bridge` | HTTP/WebSocket 调试入口，文本指令、速度控制、建图和导航调试占位 |
| `ylhb_interfaces` | `TaskEvent`、`TaskStatus`、`SayText`、`VoiceStatus` 等可复用消息 |
| `rplidar_ros-ros2` | 第三方雷达驱动 |
| `zed-ros2-wrapper` | 第三方 ZED 驱动 |

## 后续电力巡检扩展方向

后续业务可以逐步扩展为：

```text
站点/区域/路线/检查点建模
  -> 前端任务下发
  -> 机器人导航到检查点
  -> 到点语音提示
  -> 相机/云台姿态调整
  -> 图像或视频采集
  -> 检测服务识别状态和异常
  -> 后台告警和巡检记录
```

路线级检测规划包括人员、未戴安全帽、火源、烟雾和障碍物。检查点级检测规划包括开关/刀闸状态、表计/指示灯、漏油、烟火和异物。上述能力当前均为规划，尚未实现正式闭环。

## LocateAnything-3B 规划边界

LocateAnything-3B 适合作为后续检查点级复杂目标定位模型，例如到点后根据自然语言提示定位刀闸、漏油区域、异物或烟火区域。它不应替代实时避障，也不应作为当前初始化阶段的硬依赖。

当前仓库没有接入 LocateAnything-3B 推理代码，也没有声明已完成模型部署。

## LingBot-Map 规划边界

LingBot-Map 可作为后续三维空间感知模块，用于巡检视频/图片序列的三维重建、相机轨迹、点位对齐和后台三维可视化。当前阶段不把 LingBot-Map 放入机器人运动控制闭环，也没有接入实际代码。

## 后续开发路线

1. 在 Jetson/开发板上验证底盘、雷达、IMU、EKF、SLAM、Nav2。
2. 验证 ZED、YOLO/TensorRT、深度定位和感知话题。
3. 验证 `/inspection_ai/*` 任务事件、UI、语音输入和 TTS。
4. 设计正式巡检任务数据模型和 ROS 消息。
5. 增加站点、区域、路线和检查点配置。
6. 接入路线级安全检测。
7. 接入检查点级检测服务。
8. 增加告警记录、巡检记录和前端展示。
