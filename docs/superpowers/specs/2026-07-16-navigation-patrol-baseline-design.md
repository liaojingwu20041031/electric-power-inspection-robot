# 导航巡逻良好参数留档设计

## 目标

把用户现场确认“运动良好”的导航巡逻配置保存到 `docs`，便于后续调参失效时定位和恢复。

当前首份良好基线为 Git 提交 `333dbb4a09a2480b633bee71b14bf88a7e5bbc4b`。

## 方案

新增一个手动留档脚本。用户完成实机验证并确认运动良好后执行脚本，脚本在
`docs/navigation_patrol_parameter_history/` 下创建不可覆盖的时间戳目录。

同时更新 `skills/ros2-practice-first/SKILL.md`：处理 Nav2、巡逻或底盘参数调整时，若用户现场确认运动良好，必须立即调用该脚本保存基线；未获得用户确认时不得自行标记为良好版本。

每份快照保存：

- Nav2 普通与禁行区参数；
- 底盘运动学、驱动器和 EKF 参数；
- Nav2 行为树；
- 导航、禁行区导航和巡逻执行器 launch；
- 当前正式巡逻路线；
- Git 提交号、工作区状态、留档说明；
- 地图、路线和禁行区资产的 SHA256 校验值。

地图 PGM 不重复复制。它们已由 Git 管理，提交号和 SHA256 足以确认对应版本，避免每次留档重复增加大文件。

## 使用方式

```bash
scripts/save_navigation_patrol_baseline.sh "现场验证说明"
```

说明参数必填，防止产生无法判断验证内容的匿名快照。若同一秒重复执行或目标目录已存在，脚本失败且不覆盖旧记录。

## 安全边界与验证

脚本只读取和复制仓库文件，不启动 ROS2、Nav2、巡逻执行器，不发布 topic，不改变机器人状态。

静态验证只检查脚本语法、首份快照内容和校验值；实机“运动良好”结论始终由用户现场确认。

技能更新使用现有 `ros2-practice-first`，不新增重复技能或额外资源目录。
