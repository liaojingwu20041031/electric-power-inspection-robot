# 项目展示与中文手册更新设计

## 目标

将仓库 README 调整为“展示优先、开发可达”的项目入口，并让
`src/电力巡检机器人使用与调试手册.md` 成为与当前代码和运维设计一致的中文实机手册。
读者应能在 README 首屏理解机器人价值、已完成能力和安全边界；开发者应能在
不阅读整份长手册的情况下找到构建、启动和专项文档入口。

## 范围

修改：

- `README.md`
- `src/电力巡检机器人使用与调试手册.md`

复用 `记录照片/` 的实拍素材和现有 Markdown/Mermaid，不生成新图片，不引入前端
依赖，不修改机器人代码、launch、协议或运行配置。

## README 结构

1. 保留居中标题、技术徽章和实机 Hero；调整摘要为“实机机器人能力 + 安全控制边界”。
2. 在照片前增加能力快照：已完成能力、已验证范围、后续业务能力三栏，避免将规划表述为
   已交付。
3. 增加三条阅读路径：项目展示、Jetson 部署、开发扩展；各自链接到现有章节或专项文档。
4. 保留实机展示和系统架构，压缩重复叙述；明确 Agent 不直连底盘/Nav2，UI 通过
   Supervisor 管理系统生命周期。
5. 保留快速开始、构建验证、安全地图和运维入口，删除重复安全提示；将所有长篇细节继续
   链接到 `src/电力巡检机器人使用与调试手册.md` 与 `docs/` 专项文档。
6. 文档门户按“现场操作、平台连接、路线安全、三维/AI、协议与硬件资料”分类。

## 中文手册结构

1. 开头新增“当前能力快照”和“按角色阅读”索引：现场验收、机器人开发、平台/APP 联调。
2. 继续保留现有按底层到任务层的详细章节与运动安全提示，不缩减现场诊断信息。
3. 修正当前边界：Robot/Mobile Bridge 的公网 heartbeat、部署配置和协议文档已存在；
   Spring Bridge 与 Web/小程序正式业务接线仍未完成。
4. 补充本体 UI 生命周期：UI 是 inspection 全栈锚点，UI 退出主动关闭整栈；自动恢复只
   重新启动完整 inspection 栈，不允许 UI 单节点 respawn。
5. 补充 Mobile Bridge 所有权：`YLHB_MOBILE_BRIDGE_OWNER=auto` 统一解析 systemd 与
   Supervisor，避免重复实例；将核心服务、本地 APP、云平台状态分层说明。
6. 为运行类命令保持安全标记：实机运动、巡逻、Nav2 和底盘命令由现场操作员执行；文档
   不将静态验证写成实机通过。

## 事实与语言约束

- “已完成”仅用于代码、已验证流程或仓库中已存在的接口。
- “已具备基础设施”适用于 Robot Bridge heartbeat、部署配置与协议文档。
- Spring Bridge 与 Web/小程序正式业务接线统一写为“后续工作”。
- 不展示 Token、私有地址或平台环境文件内容。

## 验收

- README 内部链接和图片路径可解析，封面首屏无重复段落。
- README 的展示、部署和开发入口均在前半部分可见。
- 中文手册包含能力快照、三类阅读入口、UI 生命周期与 Mobile Bridge 所有权说明。
- 文档不改变实机启动行为，也不扩大机器人运动授权。
