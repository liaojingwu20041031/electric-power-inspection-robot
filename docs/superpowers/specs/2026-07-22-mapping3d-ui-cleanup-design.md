# 三维建模页面 UI 整理设计

## 目标

在不改变三维采集、离线重建和平台上传业务逻辑的前提下，修复长路径溢出，消除重复上传入口，并整理页面的信息层级与资源操作布局。

## 页面结构

页面按实际工作流从上到下排列：

1. 总览状态
2. 作业操作
3. 资源管理
4. 查看说明

“作业操作”包含现场采集和离线重建两个并列卡片；宽屏双列显示，窄屏上下排列。“资源管理”包含 SVO2 采集记录和重建模型记录，平台上传状态与操作只出现在对应模型卡片内部。

## 总览状态

- 保留采集状态、重建状态、最新 SVO 和最新模型四张状态卡片。
- 最新 SVO 和最新模型只显示文件名，不显示完整路径。
- 文件名直接在 QML 中从现有路径截取，不新增后端字段。
- 路径为空时继续显示“未采集”或“未生成”。

## 作业操作

- 现场采集卡片保留开始采集、停止并保存、帧数、时长和输出目录。
- 离线重建卡片保留快速重建、高质量重建、profile、输出文件和点数。
- 删除离线重建卡片中的重复“平台上传”区域。
- 不修改按钮可用条件和后端调用。

## 资源管理

### SVO2 采集记录

- 每条记录改为紧凑卡片。
- 信息区显示名称、session、帧数、时长和状态。
- 操作区显示设为最新、重命名、删除、快速重建和高质量重建。
- 信息与按钮分层排列，避免窄屏下挤成一行。

### 重建模型记录

- 保持每个模型一张卡片。
- 卡片分为模型信息、平台上传状态和资源操作三层。
- 平台上传状态继续显示状态、失败原因、重试次数和平台资产 ID。
- 上传、重试、复制 ID、设为最新、重命名和删除继续使用现有后端接口。
- 活动上传任务继续禁止重复提交和删除。

## 视觉规则

- 复用现有 `Theme`、`StatusCard`、`WarmButton` 和页面卡片样式。
- 统一标题、次要信息、状态标签和操作区的间距与字号。
- 主信息优先，session、路径、profile 和错误详情使用次级文本样式。
- 不新增页签、弹窗、筛选、分页或通用组件。

## 修改范围

- 只修改 `src/ylhb_llm/qml/pages/Mapping3DPage.qml`。
- 删除移除重复上传入口后不再使用的页面属性。
- 不修改后端、上传状态模型或 ROS2 话题。

## 验证

- 运行 `qmllint` 检查 `Mapping3DPage.qml`。
- 运行现有三维建模页面契约测试。
- 仅构建 `ylhb_llm`，关闭测试构建。
- 不新增测试文件，不运行全仓库测试。

## 验收标准

- 最新 SVO 和最新模型卡片不再因完整路径溢出。
- 页面顺序为总览、作业操作、资源管理、查看说明。
- 离线重建区不再显示平台上传入口。
- 每个模型的上传状态和操作位于对应资源卡片内。
- SVO 和模型资源在常用屏幕宽度下信息清楚、按钮不与主信息挤在同一行。
- 原有采集、重建、上传、重试、重命名、设为最新和删除行为保持不变。
