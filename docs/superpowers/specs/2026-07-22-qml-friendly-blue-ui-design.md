# QML 亲和浅蓝界面设计

## 目标

将 `src/ylhb_llm/qml` 从暖橙色操控台风格调整为普通用户易理解、易操作的智能巡检机器人应用。视觉采用浅蓝、现代、亲和的智能家居风格。

## 视觉规范

- 页面背景：`#F5FAFF`
- 卡片背景：`#FFFFFF`
- 浅蓝背景：`#EAF5FF`
- 品牌蓝：`#3B8EF3`
- 辅助天蓝：`#67C5F8`
- 主文字：`#243447`
- 辅助文字：`#7B8A9A`
- 边框：`#DDEAF4`
- 圆角：统一为 8px
- 页面间距：24px
- 控件间距：8–12px
- 卡片仅使用轻微阴影；禁止重阴影、发光和工业科技大屏效果
- 背景以 `#F5FAFF` 为底，仅在边角增加低透明度浅蓝柔和装饰，不继续使用旧背景图片

## 页面设计

### 主窗口与导航

- 保留现有侧栏结构、页面源和默认巡检页面。
- 当前导航项使用浅蓝底和品牌蓝文字突出。
- “巡逻模式”显示为“巡检任务”，“语音与 AI”显示为“AI 助手”。
- 顶部状态栏保持机器人连接状态和模式数据绑定，改用更清晰的状态胶囊样式。

### 巡检任务

- 机器人状态和任务名称保持在首屏顶部。
- 路线地图和任务控制保持现有左右布局。
- 开始巡检使用品牌蓝主按钮；暂停使用浅蓝次按钮；结束巡检保留危险色。
- 保留路线预览、平台任务确认、暂停、继续、结束、诊断、任务和事件详情的全部行为。

### AI 助手

- 使用全局浅蓝主题替代页面内独立青色主题。
- 突出语音状态、当前任务、当前步骤和最终结果。
- 保留唤醒、语音状态、Agent 状态、步骤详情和 Markdown 结果绑定。

### 其他页面

- 连接、三维建模、本机状态和日志统一卡片、文字、边框、按钮及间距。
- 不调整业务流程和数据结构。

## 功能保护边界

本次不得修改：

- `backend.*` 属性和方法名
- `onClicked`、`onToggled` 等行为中的命令和调用
- 按钮 `enabled`、页面 `visible` 等业务条件
- `pageSources` 页面映射和默认页面
- 巡检启动确认、停止确认和关机确认流程
- 路线预览的缩放、拖动、适应和重试功能

允许修改的范围仅为颜色、字体、圆角、间距、边框、轻微阴影、背景装饰、视觉层级和非协议显示文案。

## 验证

- 运行现有 QML 导航与数据绑定测试。
- 检查所有原有 `backend.*` 调用、命令字符串和页面入口仍存在。
- 运行 `ylhb_llm` 的目标构建或 QML 加载检查。
- 不新增针对颜色、固定高度、控件顺序或完整文案的脆弱测试。
