# GitHub README 项目展示优化设计

## 目标

将根目录 `README.md` 调整为蓝白工程科技风的项目展示页，使用现有机器人 Logo 和实机照片，清晰呈现机器人已经落地的硬件、导航、巡逻、感知、三维建模、AI Agent、本体 UI 与平台连接能力。

## 视觉方案

- 采用已确认的 B 方案：明亮蓝白、简洁工程名片风。
- 首屏居中展示 `记录照片/机器人卡通形象LOGO.png`，下方排列项目名称、定位、技术徽章和快速导航。
- 实机照片作为第二视觉层，保持项目真实性；不生成无关插图，不使用 GitHub 不稳定的脚本、动画或复杂 CSS。
- 统一使用蓝、青、深灰配色，依靠 HTML 表格、徽章、Mermaid 和适量图标建立层级。

## 信息结构

1. 品牌首屏：Logo、项目名称、一句话定位、技术徽章、快速导航。
2. 项目概览：用精简卡片说明自主巡逻、视觉感知、三维资产、AI Agent 和平台连接闭环。
3. 实机展示：沿用仓库已有照片，减少重复图片造成的首屏拖长。
4. 核心能力：只写仓库当前可验证的实现，不混入规划能力。
5. 系统架构：在现有 ROS 2 架构上补充 GNSS heartbeat 与三维资产上传链路。
6. 三维资产链路：说明 SVO 采集、离线 PLY 重建、自动/手动触发、SQLite、快照、SHA-256、幂等重试和平台资产 ID。
7. 快速开始、目录、开发导航、安全边界和专项文档：保留可执行入口，合并重复内容。

## 更新内容边界

README 将同步当前工作区已经实现的内容：

- 三维重建 metadata 的大小、SHA-256、格式、单位和坐标系信息；
- `/inspection_ai/scene_asset_ready` 到 `SceneUploadWorker` 的自动触发；
- 3D 页面手动上传、立即重试、上传状态、失败原因和平台资产 ID；
- SQLite 持久任务、不可变快照、单线程流式 HTTPS 上传和原幂等键重试；
- 本地 PLY/SVO 删除后清理界面残余状态；
- QML 操控台友好蓝视觉主题；
- GNSS fix、卫星数、HDOP、RTK 类型和新鲜度进入 Mobile Bridge 云状态。

不会声称 Spring/Web 审核页面已经由本仓库完成，也不会把三维点云描述为 Nav2 地图。平台审核、坐标标定和对象存储分片直传仍明确标为外部接线或后续能力。
