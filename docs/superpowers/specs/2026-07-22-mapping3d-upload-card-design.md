# 3D 建模上传卡片设计

## 目标

将“重建模型记录”从拥挤的单行列表改为友好蓝主题下的模型卡片，使模型信息、上传状态和操作按钮具有清晰层级。

## 范围

- 每个重建模型使用一张白色圆角卡片。
- 卡片上层显示名称、session、点数、文件大小和重建状态。
- 卡片下层使用浅蓝上传状态区，显示状态标签、失败原因、平台资产 ID。
- 提供上传、立即重试、重新上传、复制资产 ID、设为最新、重命名和删除操作。
- 活动上传任务继续禁止删除和重复提交。

## 最小实现

- 只修改 `Mapping3DPage.qml`，复制资产 ID 复用 Qt 剪贴板支持时才在 `ui_backend.py` 增加一个短 Slot。
- 复用 `Theme`、现有按钮和上传状态数据，不新增通用组件或全局颜色。
- 不增加弹窗、分页、筛选、预览点云和新的状态模型。
- 不新增 UI 测试文件；只运行 QML lint 和现有页面契约测试。
