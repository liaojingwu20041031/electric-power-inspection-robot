# 巡逻图片抓拍阶段修复设计

## 问题

Nav2 接受巡逻目标后，巡逻执行器仍持续发布 `navigation_phase=sending_goal`，而图片抓拍只接受 `navigation_phase=target`。因此行驶中抓拍和到点抓拍会被跳过。

## 修复

在 Nav2 目标被接受后，将当前导航阶段更新为 `target` 并立即发布状态。保留等待、发送和拒绝重试阶段的现有值，不修改 Mobile Bridge、上传协议、巡逻控制或 Nav2 行为。

## 验证

在现有巡逻执行器测试文件中增加一个回归用例：Nav2 接受目标后，状态必须立即变为 `target`。只运行该测试文件，不运行全仓库测试或实机运动。
